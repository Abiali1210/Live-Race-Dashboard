## Changes Made

### Updated `backend/src/raceState.ts`

- Added controlled mutation functions:
  - `setRaceConnectionStatus(connected)`
  - `applyNormalizedWigeMessage(message)`
- Added internal helpers:
  - `markUpdated()` sets `lastUpdate` to `new Date().toISOString()` for meaningful normalized PID updates.
  - `incrementCounter(pid)` updates `pid0`, `pid3`, `pid4`, `pid9002`, or `other`.
- `applyNormalizedWigeMessage()` behavior:
  - PID `0`: replaces `raceState.cars` with latest normalized timing snapshot.
  - PID `4`: updates `raceState.trackState`.
  - PID `3`: replaces `raceState.messages`.
  - PID `9002`: updates `raceState.stats`.
  - unknown/malformed/other PID: increments `other` only.

### Added `backend/src/wigeClient.ts`

- Exports:
  - `startWigeClient()`
  - `stopWigeClient()`
- Uses `ws` package to connect to `config.livetimingWsUrl`.
- On open:
  - sets `connected: true`
  - sends subscription payload:
    ```json
    {
      "eventId": 50,
      "eventPid": [0, 4, 3, 9002],
      "clientLocalTime": 1234567890
    }
    ```
- On message:
  - safely converts raw websocket data to UTF-8 text
  - parses JSON
  - ignores malformed/non-object payloads with a warning
  - routes through `normalizeWigeMessage()`
  - applies via `applyNormalizedWigeMessage()`
- On close:
  - sets `connected: false`
  - schedules a polite reconnect after `5000ms` if reconnecting is enabled
- On error:
  - logs a short warning and lets close/reconnect handling continue
- Includes duplicate-client guards by checking module-level `socket` and `reconnectTimer` state.

### Updated `backend/src/server.ts`

- Imports `startWigeClient`.
- Calls `startWigeClient()` inside `app.listen(...)` after starting the metadata load.
- This keeps HTTP endpoints available even if WIGE connection fails or reconnects.

## Decisions Made

- Replace arrays with latest full WIGE snapshots rather than merging because observed PID `0` sends full `RESULT` arrays.
- Keep reconnect behavior simple and polite: fixed `5000ms` delay, no exponential backoff yet.
- Keep logs useful but not noisy: startup/connect/subscribe/reconnect/errors are logged, but every inbound PID message is not logged.
- Count malformed/non-object websocket payloads as `other` via a synthetic normalized message with PID `MALFORMED`, but do not crash or persist raw payloads.
- Do not join Eventhub metadata into `TimingCar.metadata` yet; this is better handled as a later focused chunk once frontend state shape is clearer.

## Gotchas / Risks

- Live smoke testing via `Start-Job { npm start }` left an orphaned Node child process on port `3001` once during this session, causing early `/api/state` checks to hit a stale backend. Prefer `node --import tsx src/server.ts` in bounded test jobs or explicitly check/clear port `3001`.
- WIGE may be quiet at times; websocket `open` does not guarantee immediate PID messages. Poll `/api/state` until counters move.
- `other: 1` appeared in the live smoke test, likely from a non-subscribed/control PID such as `LTS_TIMESYNC`; this is expected and harmless.
- Current `connected` state only reflects websocket open/close, not freshness/staleness of recent data. A future status endpoint can expose richer diagnostics.
- Reconnect behavior is intentionally simple; exponential backoff and jitter are out of scope for Chunk 8.
- `stopWigeClient()` exists but is not yet wired into graceful process shutdown.
