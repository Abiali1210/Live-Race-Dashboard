### PID `0` timing rows

- Normalizes `RESULT` rows into `TimingCar[]`.
- Maps fields including:
  - `STNR -> carNumber`
  - `POSITION -> position`
  - `CLASSNAME -> className`
  - `LAPS -> lap`
  - `LASTLAPTIME -> lastLap`
  - `FASTESTLAP -> bestLap`
  - `GAP`/`INT -> gapToLeader`
  - `S1TIME`…`S9TIME -> sectorTimes`
  - `S1SPEED`…`S9SPEED -> sectorSpeeds`
  - `PITSTOPCOUNT -> pitStopCount`
  - `NAME -> driverName`
  - `TEAM -> teamName`
  - `CAR -> vehicle`
- Sets `metadata: null`; metadata join remains a later state/update concern.
- Sets `pitStatus: "IN"` if any sector time is `"PIT"`; otherwise `"UNKNOWN"`.
- Converts zero sector speeds such as WIGE `"0.0"` to `null` to avoid displaying misleading zero speeds for incomplete sectors.

### PID `4` track state

- Normalizes `TRACKSTATE`, `TIMESTATE`, `ENDTIME`, and `TOD` into `TrackState`.
- Preserves values as strings because semantic meanings are not yet mapped.

### PID `3` race messages

- Normalizes `MESSAGES` into `RaceMessage[]`.
- Maps `ID`, `MESSAGETIME`, `MESSAGE`, and `MESSAGEGROUP`.

### PID `9002` stats

- Validates and returns arrays for:
  - `LEADING`
  - `BESTLAPS`
  - `BESTSECTORS`
- Keeps these arrays as `unknown[]` per the existing `RaceStats` type to avoid overengineering before frontend needs are clear.

## Decisions Made

- Chunk 7 stays pure/offline: normalizers do not connect to WIGE, mutate global state, or call external services.
- Timing lap/sector times are stored as strings for now because WIGE sends formatted values such as `10:34.446`, `1:02.293`, and special values such as `PIT`.
- Track state values are preserved rather than decoded because the capture does not document the full semantic mapping.
- PID `9002` stats are shallow-normalized only, because PID `0` timing rows are the MVP priority.
- A combined `normalizeWigeMessage()` router was added now to make Chunk 8 cleaner and avoid duplicated PID routing.
