## Files Changed

- `frontend/src/api/client.ts`
  - added `DataSource = "backend" | "static"`.
  - static mode activates when `VITE_DATA_SOURCE=static` **or** Vite `MODE === "demo"`.
  - backend mode fetches:
    - `http://localhost:3001/api/status`
    - `http://localhost:3001/api/state`
  - static mode fetches:
    - `<BASE_URL>/demo-data/status.json`
    - `<BASE_URL>/demo-data/state.json`
  - added `resolveAssetUrl()` for backend/static asset path resolution.
- `frontend/src/dashboard/carHelpers.ts`
  - now uses `resolveAssetUrl()` for selected-car images.
  - static mode maps backend-like `/assets/cars/<car>.webp` paths to `<BASE_URL>/car-images/<car>.webp`.
- `frontend/vite.config.ts`
  - `mode === "demo"` sets Vite base to `/Live-Race-Dashboard/`.
- `frontend/package.json`
  - added `build:demo`: `tsc -b && vite build --mode demo`.
- `frontend/.gitignore`
  - ignores generated demo artifacts:
    - `public/car-images/`
    - `public/demo-data/`
- `backend/package.json`
  - added `export:static-demo`: `tsx src/exportStaticDemo.ts`.
- `backend/src/exportStaticDemo.ts`
  - loads Eventhub metadata snapshot.
  - loads WIGE playback once.
  - writes static snapshots to:
    - `frontend/public/demo-data/status.json`
    - `frontend/public/demo-data/state.json`
  - copies local car images from `backend/data/car-images/*.webp` to `frontend/public/car-images/*.webp`.
